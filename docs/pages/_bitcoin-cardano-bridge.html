Bitcoin on Cardano
without Bitcoin moving

How Taproot, BitVMX and Cardano smart contracts let you use Bitcoin's value across two blockchains — while your BTC never leaves Bitcoin's network. A plain-language guide with interactive simulators.

Overview
The big idea
Bitcoin is the most trusted store of value on the internet. Cardano is a programmable smart-contract platform. What if you could use your Bitcoin's value on Cardano — earning yield, trading, lending — while your actual BTC never leaves the safety of the Bitcoin network?
Think of it like a safety deposit box with a digital window. You put your gold in the vault (Bitcoin), and a screen on the street (Cardano) shows a live certificate proving the gold is there. Anyone can trade the certificate. When you want your gold back, you tear up the certificate and the vault opens automatically — no staff, no middleman, no trust required.
🔒

Bitcoin stays put

Your BTC is locked in a special vault on the Bitcoin blockchain. It cannot move unless specific conditions are met — provably, mathematically.

🪙

A receipt on Cardano

A token (cBTC) is minted on Cardano representing your locked BTC. You can move, trade, or use it freely. It's backed 1:1 by real BTC in the vault.

✅

Cryptographic proof, not trust

No company holds your BTC. The lock is enforced by Bitcoin's own script rules. Cardano checks a mathematical proof before minting or burning any token.

🔁

Redeem any time

Burn the Cardano token, submit proof to Bitcoin, and your BTC unlocks back to your wallet. The system runs perfectly in both directions.

Layer 1 — Bitcoin
Taproot — Bitcoin's secret tree of rules
Activated on Bitcoin in November 2021, Taproot lets you encode complex spending conditions into a single address — and only reveal the rule you actually used. The address looks completely normal from the outside.
What is Taproot?
Before Taproot, if you wanted Bitcoin locked by complex rules, the entire rulebook was visible on-chain for anyone to inspect. Taproot uses a clever maths trick — combining Merkle trees with Schnorr signatures — to hide all the rules inside a single public key. The address looks like any ordinary Bitcoin address. Only when you spend the coins do you reveal just the one rule you actually used — the others remain private forever.
Taproot address (looks completely normal)
▼ secretly encodes a Merkle tree of scripts inside the key
Script A — revealed when used BitVMX bridge unlock: valid Cardano burn proof + operator co-signature
Script B — permanently hidden Emergency: 2-of-3 federation after 6 months of operator inactivity
Script C — permanently hidden Key-path spend: instant release if all parties agree simultaneously
Only Script A is ever revealed on-chain. B and C remain private forever — even after the coins move.

Why does this matter for bridges?

The vault encodes all unlock conditions — normal release, emergency recovery, federation override — but looks like a simple address. This dramatically reduces on-chain cost and hides the bridge mechanics from observers.

Schnorr signatures

Taproot uses Schnorr signatures, which can be aggregated. Multiple bridge operators each sign independently and their signatures combine into a single compact signature — indistinguishable from a solo spend.

The vault in plain English
When you send BTC to the bridge, it goes to a Pay-to-Taproot (P2TR) address. That address secretly contains the rule: "These coins can only move if someone proves on Cardano that the wrapped token was burned." Without that proof, the coins are mathematically immovable — not because someone is guarding them, but because the cryptography makes it impossible.
The bridge engine
BitVMX — running programs inside Bitcoin
Bitcoin was intentionally designed to be simple and not run complex programs. BitVMX is an ingenious workaround: run computations off-chain, then use Bitcoin scripts to verify the result — only settling disputes on-chain when someone cheats.
The challenge
Cardano can run sophisticated smart contracts. Bitcoin cannot. For a trustless bridge, Bitcoin needs to verify that something happened on Cardano — specifically, that a wrapped token was burned. BitVMX makes this possible without changing a single rule of the Bitcoin protocol. The simulator tab shows the actual RISC-V computation in detail.
⚡

Optimistic execution

Everyone assumes the bridge operator is honest. No expensive proof is needed in the normal case. The heavy machinery only activates if someone tries to cheat.

🎮

RISC-V challenge game

If an operator cheats, a watchdog triggers a dispute. The operator must execute a RISC-V program step-by-step using Bitcoin scripts — one wrong step and they lose their bond.

🧮

Verifiable computation

The BitVMX RISC-V CPU can verify any computation — including "did this Cardano block really contain a valid burn?" — using Bitcoin's own script opcodes.

🛡️

Economic security

Operators must post a collateral bond. If caught cheating, they lose their bond entirely. Honest behaviour is always more profitable than cheating.

What is a Plutus minting policy?
Cardano uses a smart-contract language called Plutus. A "minting policy" is a small program that controls when a new token can be created. The bridge's minting policy checks three things before allowing cBTC to be minted: (1) a valid Bitcoin lock transaction exists on-chain, (2) the amount locked matches the amount being minted, and (3) the Bitcoin vault address matches the bridge's vault. Only if all three pass does cBTC come into existence.
📜

eUTxO model

Each cBTC token carries a datum — data embedded in the transaction — including the Bitcoin txid that backs it. This makes auditing completely transparent and verifiable by anyone.

🔍

On-chain audit trail

Every mint and every burn is permanently recorded on Cardano. Anyone can verify at any moment that total cBTC supply exactly equals total BTC locked in the vault.

Common questions
Things people ask
Plain answers to the most common questions about wrapped Bitcoin bridges and how they stay trustless.
The vault is a Taproot address. The only way to move the BTC is to satisfy one of the scripts encoded in the Taproot tree. The primary script requires a valid burn proof from Cardano plus a co-signature from the bridge operator. The operator's collateral bond makes cheating economically irrational — they'd lose more than they'd gain. BitVMX is the mathematical backstop: even if an operator tried to steal, a watchdog node can trigger the on-chain RISC-V verification and force the funds back to the user.
The Taproot script tree includes an emergency recovery path hidden in Script B. After a time-lock period — say, 6 months of inactivity — a federation of independent parties can co-sign to release funds. No single party can block access to your BTC indefinitely.
No — that's the elegance of BitVMX. It works entirely within Bitcoin's existing script rules, using only Taproot (live since November 2021). No new opcodes, no soft forks, no waiting for protocol changes. The innovation is entirely in how the scripts are structured and how the off-chain RISC-V challenge game is designed.
RISC-V is an open-source, simple instruction set architecture — essentially a very simple CPU design. BitVMX uses it because its instructions are small enough to be encoded into Bitcoin scripts one-by-one. This means any computation (including "verify this Cardano block header and transaction Merkle proof") can be broken into steps that Bitcoin can verify in a dispute. The verifier program runs in RISC-V, not Bitcoin Script directly.
WBTC (Wrapped Bitcoin on Ethereum) is custodial — a company called BitGo physically holds your BTC and you trust them not to run off with it, get hacked, or face regulatory shutdown. The Taproot + BitVMX approach is trustless: the conditions are enforced by Bitcoin's own cryptography. There is no company that "holds" your BTC. The difference is trusting a company versus trusting mathematics.
Wrapping (BTC → cBTC) requires 6 Bitcoin block confirmations — about 60 minutes — for security. Unwrapping (cBTC → BTC) in the happy path takes roughly 1 Bitcoin confirmation after the operator releases. The BitVMX dispute path would take longer, but is expected to rarely if ever be needed in practice.
cBTC is a native Cardano token (CIP-25 standard) minted by the bridge's Plutus policy. It behaves like any Cardano token — trade on DEXes like Minswap or SundaeSwap, supply to lending protocols, provide liquidity, use as collateral, or simply send to another wallet. The only special property: burning it triggers the BTC release from the vault.
